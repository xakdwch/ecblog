# 背景

etcd的原始接口只能按照Key的前缀来批量同步数据，源集群中指定前缀下的所有数据都将被同步到目的集群。然而在某些场景下，我们期望有更细粒度的规则来精准地同步数据。我们可以通过配置Etcd-Carry过滤规则来达到这样的效果💯。

提示

当前过滤规则主要适用于过滤etcd中的Kubernetes数据。
